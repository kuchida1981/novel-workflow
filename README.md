# Novel Workflow

小説執筆にソフトウェア開発のエンジニアリング・プラクティスを適用するためのフレームワークおよび執筆レポジトリです。

## 概要

このプロジェクトは、物語の「核（テーマ）」を最上流の制約とし、素材（キャラクター・世界観）、構造（プロット）、仕様（シーン設計）を経て本文（ドラフト）へと落とし込んでいくライフサイクルを管理します。

## ワークフローの構造

1. **Step 1: 核 (Core)** - 物語の根幹、テーマ、問いの定義。
2. **Step 2: 素材 (Assets)** - キャラクタープロファイルと世界観設定。
3. **Step 3: 構造 (Plot)** - 三幕構成や起承転結による骨格設計。
4. **Step 4: シーン (Scenes)** - 各章の具体的な執筆仕様。
5. **Step 5: 本文 (Draft)** - 実際の執筆。
6. **振り返り (Post-mortem)** - プロジェクトの分析と改善。
