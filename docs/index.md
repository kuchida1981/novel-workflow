# Novel Workflow (NW) 総合ガイド

Novel Workflow (NW) は、ソフトウェアエンジニアリングの規律を物語執筆に持ち込み、AIとの協調作業によって一貫性と深度のある作品を創り出すためのフレームワークです。

## 1. 基本思想

### 制約による自由
物語の根幹（Core）を「仕様」として定義し、それを下流工程の絶対的な制約とすることで、執筆中の迷いを排除します。制約があるからこそ、その範囲内で最大限の創造力を発揮できます。

### 会計的な物語構築
キャラクターの行動や世界の出来事を、因果の「収支」として捉えます。誰かの幸運には必ず誰かの代償が伴うという「帳簿」の視点が、物語に冷徹なリアリティと緊張感を与えます。

### 承認ゲートによる合意形成
AI（エージェント）と人間（ユーザー）の間で、各ステップごとに明示的な承認を行うことで、イメージの乖離を防ぎ、一歩ずつ確実に完成へと近づけます。

## 2. ディレクトリ構造と役割

- **`README.md`**: プロジェクト全体の入り口。
- **`docs/`**: フレームワーク自体のドキュメント。
  - `index.md`: このガイド。全体像と哲学。
  - `usage-example.md`: 実践的な対話と執筆のケーススタディ。
- **`skeleton/`**: 新しい物語を開始するためのテンプレート。
  - `docs/workflow.md`: 各プロジェクト内で参照する作業マニュアル。
- **`projects/`**: 執筆中のプロジェクト。
- **`projects/archive/`**: 完了したプロジェクトの保存先。

## 3. ライフサイクルの流れ

1. **Research & Strategy**: テーマの探索と、NWの適用。
2. **Step 1: Core (核)**: テーマ、トーン、非目標の定義。
3. **Step 2: Assets (素材)**: キャラクターと世界観の設計。
4. **Step 3: Plot (構造)**: 幕構成による骨格の構築。
5. **Step 4: Scenes (仕様)**: 章をシーン単位に分解。
6. **Step 5: Draft (本文)**: シーン仕様に基づく執筆。
7. **Review (振り返り)**: ポストモーテムによる改善点の抽出。

## 4. 運用プロトコル

### 承認リクエスト・プロトコル
エージェントは成果物を提示するたびに自発的に承認を求めます。ユーザーの承認なしに次のステップへ進むことはありません。

### アーカイブ・プロトコル
完了したプロジェクトは `archive/` ディレクトリへ移動され、`projects/` 直下は常にアクティブな作業空間として保たれます。

## 5. 実践に向けて

初めての方は、[実践例：『誰の身代わりでもなく』執筆プロセス](./usage-example.md) を一読することをお勧めします。AIとの具体的な対話のテンポや、リアリティを向上させるための軌道修正の要点がまとめられています。
